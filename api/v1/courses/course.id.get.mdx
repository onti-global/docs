# GET Course

<Warning>Planned — this endpoint is not implemented in the backend yet.</Warning>

Returns a single published course by id.

## Authentication

- Requires a valid Supabase Auth session (`Authorization: Bearer <token>`).
- Uses a Supabase client scoped to the caller's JWT; all reads and writes are enforced by Row Level Security (RLS).
- Returns `404` for any course that is not `status = "published"` (draft and archived courses are invisible to learners).

## Headers

| Header | Value | Required |
| --- | --- | --- |
| `Authorization` | `Bearer <token>` | Yes |
| `Accept` | `application/json` | Yes |

## Path parameters

| Name | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | The course id. |

## Response body

`200 OK`

```json
{
  "status": 200,
  "message": "",
  "data": {
    "id": "1aa2...d4",
    "title": "Foundations of Coaching",
    "description": "An introduction to core coaching frameworks.",
    "status": "published",
    "created_at": "2026-01-15T09:00:00Z",
    "updated_at": "2026-01-15T09:00:00Z"
  }
}
```

### Fields

| Field | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | Course id. |
| `title` | `string` | Course title. |
| `description` | `string \| null` | Optional marketing description. |
| `status` | `"published"` | Always `"published"` in catalog responses. |
| `created_at` | `timestamptz` | When the course was created. |
| `updated_at` | `timestamptz` | When the course was last modified. |

## Errors

| Status | Code | When |
| --- | --- | --- |
| `401` | `unauthenticated` | Missing or invalid JWT. |
| `404` | `not_found` | Course does not exist or is not published. |
| `500` | `internal_error` | Unexpected database failure. |

```json
{
  "status": 404,
  "message": "Not found",
  "error": "course not found",
  "data": null
}
```
