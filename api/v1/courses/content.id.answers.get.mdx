# GET Content Answers

Returns the authenticated learner's submitted answers for a content item. Returns `404` if no submission exists yet.

## Authentication

- Requires a valid Supabase Auth session (`Authorization: Bearer <token>`).
- Uses a Supabase client scoped to the caller's JWT; all reads and writes are enforced by Row Level Security (RLS).
- The learner must be assigned to the variant that contains this content item.

## Headers

| Header | Value | Required |
| --- | --- | --- |
| `Authorization` | `Bearer <token>` | Yes |
| `Accept` | `application/json` | Yes |

## Path parameters

| Name | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | The content item id. |

## Response body

`200 OK`

```json
{
  "status": 200,
  "message": "",
  "data": {
    "content_id": "c2...gg",
    "answers": {
      "question_1": "option_b",
      "question_2": ["option_a", "option_c"]
    },
    "submitted_at": "2026-06-29T11:30:00Z"
  }
}
```

| Field | Type | Description |
| --- | --- | --- |
| `content_id` | `uuid` | The content item id. |
| `answers` | `object` | The learner's most recent answers. Shape is defined by the content item's `body` JSON schema. |
| `submitted_at` | `timestamptz` | When the answers were last submitted. |

## Errors

| Status | Code | When |
| --- | --- | --- |
| `401` | `unauthenticated` | Missing or invalid JWT. |
| `403` | `variant_not_assigned` | The learner is not assigned to the variant containing this content. |
| `404` | `content_not_found` | Content item does not exist. |
| `404` | `no_submission` | The learner has not submitted answers for this item yet. |
| `500` | `internal_error` | Unexpected database failure. |

```json
{
  "status": 404,
  "message": "Not found",
  "error": "no answers submitted for this content item",
  "data": null
}
```
