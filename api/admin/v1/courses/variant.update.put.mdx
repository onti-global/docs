# PUT Update Variant

Updates a variant's name or description. All fields are optional — only provided fields are updated.

## Authentication

- Requires a valid Supabase Auth session with `admin_dashboard_role = "admin"` or `"analyst"` in the JWT.
- Uses the Supabase **admin** client (service-role key); requests bypass RLS entirely.

## Headers

| Header | Value | Required |
| --- | --- | --- |
| `Authorization` | `Bearer <token>` | Yes |
| `Content-Type` | `application/json` | Yes |
| `Accept` | `application/json` | Yes |

## Path parameters

| Name | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | The variant id. |

## Request body

```json
{
  "name": "High-Autonomy Path (Revised)",
  "description": "Updated description."
}
```

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `name` | `string` | No | New variant name. Must be non-empty if provided. |
| `description` | `string \| null` | No | New description. Pass `null` to clear it. |

## Response body

`200 OK` — returns the updated variant.

```json
{
  "status": 200,
  "message": "Variant updated",
  "data": {
    "id": "v3...ii",
    "module_id": "m3...hh",
    "name": "High-Autonomy Path (Revised)",
    "description": "Updated description.",
    "created_at": "2026-06-29T12:15:00Z",
    "updated_at": "2026-06-29T13:00:00Z"
  }
}
```

| Field | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | Variant id. |
| `module_id` | `uuid` | Parent module id. |
| `name` | `string` | Variant name. |
| `description` | `string \| null` | Variant description. |
| `created_at` | `timestamptz` | Row creation timestamp. |
| `updated_at` | `timestamptz` | Row last-modified timestamp. |

## Errors

| Status | Code | When |
| --- | --- | --- |
| `401` | `unauthenticated` | Missing or invalid JWT. |
| `403` | `forbidden` | JWT does not carry an allowed `admin_dashboard_role` (`admin` or `analyst`). |
| `400` | `validation_error` | A provided field value is invalid (e.g. empty name). |
| `404` | `variant_not_found` | Variant does not exist. |
| `500` | `internal_error` | Unexpected database failure. |

```json
{
  "status": 400,
  "message": "name must be a non-empty string",
  "error": "validation_error",
  "data": null
}
```
