# PATCH Update Enrollment (Admin)

Updates the status of an existing enrollment. Use this to manually mark a learner as completed, re-activate a dropped learner, or drop a learner administratively. `completed_at` is set automatically by the server when `status` transitions to `"completed"`, and reset to `null` when it transitions to any other status.

## Authentication

- Requires a valid Supabase Auth session with `admin_dashboard_role = "admin"` or `"analyst"` in the JWT.
- Uses the Supabase **admin** client (service-role key); requests bypass RLS entirely.

## Headers

| Header | Value | Required |
| --- | --- | --- |
| `Authorization` | `Bearer <token>` | Yes |
| `Content-Type` | `application/json` | Yes |
| `Accept` | `application/json` | Yes |

## Path parameters

| Name | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | The enrollment id. |

## Request body

```json
{
  "status": "completed"
}
```

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `status` | `"active" \| "completed" \| "dropped"` | Yes | New enrollment status. |

## Errors

| Status | Code | When |
| --- | --- | --- |
| `401` | `unauthenticated` | Missing or invalid JWT. |
| `403` | `forbidden` | JWT does not carry an allowed `admin_dashboard_role` (`admin` or `analyst`). |
| `400` | `validation_error` | `status` is missing or not a valid enum value. |
| `404` | `enrollment_not_found` | Enrollment does not exist. |
| `500` | `internal_error` | Unexpected database failure. |

```json
{
  "status": 400,
  "message": "status must be one of: active, completed, dropped",
  "error": "validation_error",
  "data": null
}
```
