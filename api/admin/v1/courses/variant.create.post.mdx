# POST Create Variant

Adds a new variant to a module. Variants are the personalised paths through a module — a module must have at least one variant before content can be added. Multiple variants allow the system to assign different experiences to different learners.

## Authentication

- Requires a valid Supabase Auth session with `admin_dashboard_role = "admin"` in the JWT.
- Uses the Supabase **admin** client (service-role key); requests bypass RLS entirely.

## Headers

| Header | Value | Required |
| --- | --- | --- |
| `Authorization` | `Bearer <token>` | Yes |
| `Content-Type` | `application/json` | Yes |
| `Accept` | `application/json` | Yes |

## Path parameters

| Name | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | The parent module id. |

## Request body

```json
{
  "name": "High-Autonomy Path",
  "description": "For learners with prior facilitation experience."
}
```

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `name` | `string` | Yes | Variant name. Used by admins to distinguish paths (not shown to learners). |
| `description` | `string` | No | Optional internal description. |

## Response body

`201 Created`

```json
{
  "id": "v3...ii",
  "module_id": "m3...hh",
  "name": "High-Autonomy Path",
  "description": "For learners with prior facilitation experience.",
  "created_at": "2026-06-29T12:15:00Z",
  "updated_at": "2026-06-29T12:15:00Z"
}
```

| Field | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | Variant id. |
| `module_id` | `uuid` | Parent module id. |
| `name` | `string` | Variant name. |
| `description` | `string \| null` | Variant description. |
| `created_at` | `timestamptz` | Row creation timestamp. |
| `updated_at` | `timestamptz` | Row last-modified timestamp. |

## Errors

| Status | Code | When |
| --- | --- | --- |
| `401` | `unauthenticated` | Missing or invalid JWT. |
| `403` | `forbidden` | JWT does not carry `admin_dashboard_role = "admin"`. |
| `400` | `invalid_body` | `name` is missing or empty. |
| `404` | `module_not_found` | Module does not exist. |
| `500` | `internal_error` | Unexpected database failure. |

```json
{
  "error": {
    "code": "module_not_found",
    "message": "module not found"
  }
}
```
