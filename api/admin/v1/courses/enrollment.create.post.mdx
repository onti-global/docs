# POST Create Enrollment (Admin)

Manually enrolls a user in a course, bypassing entitlement checks. Used for admin-provisioned access (comps, gifted access, bulk onboarding). The operation is an idempotent upsert — enrolling an already-enrolled user returns the existing enrollment.

## Authentication

- Requires a valid Supabase Auth session with `admin_dashboard_role = "admin"` or `"analyst"` in the JWT.
- Uses the Supabase **admin** client (service-role key); requests bypass RLS entirely.

## Headers

| Header | Value | Required |
| --- | --- | --- |
| `Authorization` | `Bearer <token>` | Yes |
| `Content-Type` | `application/json` | Yes |
| `Accept` | `application/json` | Yes |

## Request body

```json
{
  "user_id": "9c0a...77",
  "course_id": "1aa2...d4"
}
```

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `user_id` | `uuid` | Yes | The user to enroll. Must exist in `auth.users`. |
| `course_id` | `uuid` | Yes | The course to enroll the user in. |

## Response body

`201 Created`

```json
{
  "status": 200,
  "message": "Enrollment created",
  "data": {
    "id": "e9f0...12",
    "user_id": "9c0a...77",
    "course_id": "1aa2...d4",
    "status": "active",
    "enrolled_at": "2026-06-29T10:00:00Z",
    "completed_at": null,
    "created_at": "2026-06-29T10:00:00Z",
    "updated_at": "2026-06-29T10:00:00Z",
    "user": {
      "email": "ana@example.com",
      "display_name": "Ana García"
    }
  }
}
```

| Field | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | Enrollment id. |
| `user_id` | `uuid` | Enrolled user id. |
| `course_id` | `uuid` | Course id. |
| `status` | `"active" \| "completed" \| "dropped"` | Enrollment lifecycle state. |
| `enrolled_at` | `timestamptz` | When the user was enrolled. |
| `completed_at` | `timestamptz \| null` | When completion was recorded. |
| `created_at` | `timestamptz` | Row creation timestamp. |
| `updated_at` | `timestamptz` | Row last-modified timestamp. |
| `user` | `object` | Basic user identity: `{ email: string \| null, display_name: string \| null }`. |

## Errors

| Status | Code | When |
| --- | --- | --- |
| `401` | `unauthenticated` | Missing or invalid JWT. |
| `403` | `forbidden` | JWT does not carry an allowed `admin_dashboard_role` (`admin` or `analyst`). |
| `400` | `validation_error` | `user_id` or `course_id` is missing or not a valid UUID. |
| `404` | `user_not_found` | No user with that id exists. |
| `404` | `course_not_found` | Course does not exist. |
| `500` | `internal_error` | Unexpected database failure. |

```json
{
  "status": 404,
  "message": "No user found with the provided user_id",
  "error": "user_not_found",
  "data": null
}
```
