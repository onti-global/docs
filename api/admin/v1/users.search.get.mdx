# GET Search Users

Searches users by email for the enrollment picker. The match is a case-insensitive *contains* on the email address. Returns at most 15 results and excludes soft-deleted users. An empty or missing `query` returns an empty list.

## Authentication

- Requires a valid Supabase Auth session with `admin_dashboard_role = "admin"` or `"analyst"` in the JWT.
- Uses the Supabase **admin** client (service-role key); requests bypass RLS entirely.

## Headers

| Header | Value | Required |
| --- | --- | --- |
| `Authorization` | `Bearer <token>` | Yes |
| `Accept` | `application/json` | Yes |

## Query parameters

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `query` | `string` | No | Text to match against user emails (case-insensitive, contains). Empty or missing returns `items: []`. |

## Response body

`200 OK`

```json
{
  "status": 200,
  "message": "",
  "data": {
    "items": [
      {
        "id": "9c0a...77",
        "email": "ana@example.com",
        "display_name": "Ana García"
      }
    ]
  }
}
```

| Field | Type | Description |
| --- | --- | --- |
| `items` | `UserSummary[]` | Matching users. Max 15 results. Soft-deleted users are excluded. |

### UserSummary object

| Field | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | User id. |
| `email` | `string \| null` | User email. |
| `display_name` | `string \| null` | User display name. |

## Errors

| Status | Code | When |
| --- | --- | --- |
| `401` | `unauthenticated` | Missing or invalid JWT. |
| `403` | `forbidden` | JWT does not carry an allowed `admin_dashboard_role` (`admin` or `analyst`). |
| `500` | `internal_error` | Unexpected database failure. |

```json
{
  "status": 401,
  "message": "Missing or invalid authorization token",
  "error": "unauthenticated",
  "data": null
}
```
