# GET User Products

Provides the Apps with the product-level access a user has based on their purchases and subscription level. Backed by the `user_products` table — one row per (user, product) pair, where a product is exactly one of a course, an agent, or an onti package.

## Authentication

- Requires a valid Supabase Auth session (JWT in the `Authorization: Bearer <token>` header).
- The endpoint returns products for the authenticated user only. The user id is taken from the JWT (`auth.uid()`), never from a request parameter.

## Query parameters

| Name | Type | Required | Description |  |  |
| --- | --- | --- | --- | --- | --- |
| `product_kind` | \`"course" | "agent" | "onti\_package"\` | No | Filter results to a single product kind. Omit to return all kinds. |
| `status` | \`"active" | "expired" | "revoked"\` | No | Filter by entitlement status. Defaults to `"active"` so apps get only currently usable products. |
| `include` | `"product"` (comma-separated, repeatable) | No | Hydrate the referenced product object (course / agent / onti\_package) inline instead of id-only. |  |  |

### Row fields

| Field | Type | Description |  |  |
| --- | --- | --- | --- | --- |
| `id` | `uuid` | Entitlement row id (`user_products.id`). |  |  |
| `user_id` | `uuid` | Authenticated user id (`auth.users.id`). |  |  |
| `product_kind` | \`"course" | "agent" | "onti\_package"\` | Which product type this row grants. Exactly one of the `*_id` fields below will be non-null. |
| `course_id` | \`uuid | null\` | Set when `product_kind = "course"`. |  |
| `agent_id` | \`uuid | null\` | Set when `product_kind = "agent"`. |  |
| `package_id` | \`uuid | null\` | Set when `product_kind = "onti_package"`. |  |
| `status` | \`"active" | "expired" | "revoked"\` | Current entitlement state. |
| `granted_at` | `timestamptz` | When access was granted. |  |  |
| `expires_at` | \`timestamptz | null\` | End of access window. `null` means no expiration (typical for one-time course purchases). |  |
| `product` | \`object | undefined\` | Only present when `include=product` is requested. Shape varies by `product_kind` (see above). |  |

## Errors

| Status | Code | When |
| --- | --- | --- |
| `401` | `unauthenticated` | Missing or invalid JWT. |
| `400` | `invalid_query_param` | Unknown `product_kind`, `status`, or `include` value. |
| `500` | `internal_error` | Unexpected failure reading from the database. |

```json
{
  "error": {
    "code": "invalid_query_param",
    "message": "product_kind must be one of: course, agent, onti_package"
  }
}
```

## Notes

- The endpoint reflects entitlements only. It does not return progress, conversation history, or any per-product state — those belong to the per-product services.
- Apps should treat `status = "active"` plus (no `expires_at` or `expires_at > now()`) as the only sufficient condition for granting access.
- The schema guarantees exactly one of `course_id` / `agent_id` / `package_id` is non-null per row, so apps can switch on `product_kind` safely.
